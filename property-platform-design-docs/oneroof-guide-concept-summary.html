OneRoof Guide
Concept Summary · Design Challenge
Mari Kondrateva · April 2026
An agentic search experience for property discovery — mobile-first
Built as a functional HTML prototype rather than a static Figma mock — the workflow described in "Operating as an AI-native product designer in 2026". With Claude Design (launched 17 April) reading a team's codebase and design system directly, this is rapidly becoming how designers will ship.

The Problem

OneRoof's current app offers 20+ filters, suburb profiles, estimates, school zones, commute times and council data — but each lives in a separate tool. Users translate real questions ("near a good school, 30 min to work, not on a busy road") into checkboxes, then cross-reference results manually.

App Store reviews confirm the gap: users ask for keyword search ("do-up", "urgent sale"), complain "For You" can't be filtered, and wish saved searches remembered what matters to them. The data is rich. The intelligence connecting it isn't.

The Opportunity

No major NZ property portal offers AI-powered search. Trade Me is a generalist marketplace with filter-only search. Realestate.co.nz is industry-owned with strong map search but no AI. Homes.co.nz is a data reference tool, not listings-first.

Internationally, Zillow launched natural language search in 2023, ChatGPT integration in Oct 2025, and full conversational AI Mode in Mar 2026. Homes.com and Realtor.com followed. In NZ, the market is wide open — and OneRoof holds the strongest data foundation to move first.

Who We're Designing For

Primary users: First-home buyers — anxious, often uneducated on the process, highest emotional and financial stakes. Move-up buyers — time-pressed, already own and need to coordinate buy and sell. Investors — data-driven, yield-focused, fast to shortlist.

Secondary users: Real estate agents (OneRoof's paying customers — they need qualified leads, not volume). Sellers, renters, and property-curious homeowners who rely on OneRoof Estimates and suburb data.

The Concept — OneRoof Guide

A mobile-first agentic search layer that sits alongside existing filter search. Meets users where they are — whether starting out or deep in a months-long search — with three modes of interaction:

Ask
Describe what you want in your own words. The Guide translates intent into a multi-dimensional query and returns ranked matches with "Why this matches" reasoning.
Learn
As users browse, save and dismiss, the Guide learns implicit preferences and surfaces them back with consent. Delivers timely nudges: price drops, new matches, market shifts.
Guide
Structures trade-offs when comparing shortlisted properties. Brings estimates, suburb trends, commute and mortgage context into a single decision view — without deciding for the user.

Rollout

P1 (3mo): NL search, Router + Search Specialist, 10% cohort · P2 (3mo): Insights cards, all users · P3 (3mo): For You nudges, comparison, learned preferences, desktop · P4: Coach Specialist for first-home buyers, agent-facing tools.

Technical Architecture — Multi-Agent System

User input → Router (Haiku/Flash) → Specialist (Sonnet)

Router agent (lightweight, cheap, fast) accepts input, classifies intent, dispatches to a specialist. It streams transparency messages on-screen ("Understanding your needs…", "Searching 46,000+ listings…") while the specialist works in parallel. Messages build trust and mask latency.

Specialist agents (scoped, prompt-tuned): Search · Insights · Comparison · Coach. Each handles one capability well. Swappable, individually observable, invoked only when justified.

What This Means for OneRoof's Design System

The Guide reuses the existing system wherever possible — cards, list rows, pill buttons, tab bar, type, spacing. What's new is small: a Guide tab icon, a match-% badge, reasoning tags (green ✓ / amber ⚠ / blue ↗), a confidence chip, a streamed Router status line, and a "Why this matches" dialog. Roughly one atom, two molecules, one organism — the rest is composition.

With the OneRoof codebase and Figma library in hand, Claude Design (Anthropic Labs, launched 17 April) would generate these against the real tokens — shortening concept-to-developer-ready prototype from weeks to days, and how I'd expect to work on this team.

Business Model & Commercial Considerations

Free for end users. Property search is top-of-funnel. Gating it behind a paywall hands differentiation to Trade Me the moment they catch up. Monetisation flows through existing revenue streams, amplified: better-qualified enquiries justify premium agent listings and subscriptions.

Cost containment. Router-first pattern + caching of common suburb/school/commute data keeps per-query cost in single-digit NZ cents. Heavy specialists run only when justified.

Future revenue: agent-facing AI tools, labelled sponsored suggestions for new-builds, home-loan partner integration when the Guide discusses affordability. What we avoid: charging end users, hiding listings behind AI, compromising Guide neutrality.

Key Metrics & KPIs

Engagement Time-to-shortlist (target −30%); weekly active on "For You"; search-to-enquiry conversion; return sessions per saved property
Lead Quality Enquiry-to-viewing rate (agent-reported); agent NPS on leads (+15pts goal); impression-to-enquiry ratio vs. filter search
Trust & Health Suggestion acceptance (40–70%); reversion to filter search (<15%); explainability click-through; negative feedback (<5%)

Testing & Validation

Pre-launch eval. ~200 real NZ queries (from saved-search logs + App Store review language) with expected top-5 results, scored by property experts. Every agent change must pass before shipping. LLM-as-judge for hallucination and citation coverage.

Moderated usability testing — 6–8 users per segment (first-home, move-up, investor) each phase. Can they articulate what they want? Do they trust the reasoning? Recruited from existing saved-search users.

In-market A/B — 10% cohort at P1, Guide vs. filter-search control. Primary metric: search-to-enquiry rate. Guardrails: reversion, negative feedback, latency. Quarterly qualitative check-ins with 10 agencies validate lead quality.
